📂 Phase 1. 기획 및 구조 설계 (FigJam & HTML 기초)
가장 먼저 뼈대를 잡는 단계입니다.

"부산 여행 가이드 웹 서비스를 만들려고 해. 사용자가 질문하면 공공데이터를 기반으로 추천해주는 서비스의 핵심 기능 3가지를 제안해줘."

"이 서비스의 사용자 흐름도(User Flow)를 텍스트로 그려줘. (입력 -> API 호출 -> AI 처리 -> 출력)"

"HTML5 기본 뼈대 코드를 작성해줘. 모바일 뷰포트 설정과 한글 폰트(Noto Sans KR) CDN을 포함해줘."

"화면 구성을 '상단 헤더', '중앙 채팅창', '하단 입력창' 3단 구조로 나누려고 해. 기본적인 HTML 구조를 잡아줘."

"사용자가 입력한 텍스트를 감지하고 전송 버튼을 누르면 콘솔에 출력하는 자바스크립트 기본 함수를 만들어줘."

📂 Phase 2. 공공데이터 연동 (Data Integration)
부산시 API 3개를 동시에 불러오고 데이터를 정리하는 단계입니다.

"Javascript의 fetch 함수를 사용해서 공공데이터포털의 JSON 데이터를 불러오는 예시 코드를 보여줘."

"부산 명소, 맛집, 축제 3개의 API 주소가 있어. Promise.all을 사용해서 이 3개를 동시에 호출하고 로딩 시간을 단축하는 코드를 짜줘."

"API에서 받아온 데이터가 너무 복잡해. 여기서 '장소명(MAIN_TITLE)', '위치(GUGUN_NM)', '설명(ITEMCNTNCN)'만 뽑아서 새로운 배열로 정리하는 코드를 작성해줘."

"이미지를 제외하고 텍스트 데이터만 가볍게 처리하고 싶어. 데이터 파싱 과정에서 불필요한 필드를 제거해줘."

"데이터 로딩이 완료되면 채팅창에 '데이터 통합 완료'라는 시스템 메시지를 띄우는 기능을 추가해줘."

"API 키가 노출되지 않도록 변수(const DATA_GO_KR_KEY)로 따로 관리하고 싶어. 코드를 수정해줘."

📂 Phase 3. 생성형 AI (Gemini) 연동
가장 중요한 AI 두뇌를 연결하는 단계입니다.

"Google Gemini API를 REST API 방식으로 호출하는 자바스크립트 fetch 코드를 알려줘."

"사용자의 질문과 내가 수집한 '부산 공공데이터 배열'을 합쳐서 프롬프트로 만드는 방법을 알려줘."

"시스템 프롬프트에 '너는 부산 여행 전문 가이드야'라는 페르소나를 부여하고 싶어. 프롬프트 내용을 작성해줘."

"AI가 답변할 때 '[맛집] 장소명' 같은 형식을 지키도록 지시사항을 프롬프트에 추가해줘."

"Gemini API 호출 시 404 Not Found 에러가 발생해. 모델명을 gemini-1.5-flash에서 최신 모델로 변경해야 할까?" (중요: 디버깅 과정)

"내 API 키에서 사용 가능한 모델 리스트를 조회하는 URL을 알려줘."

"모델명을 gemini-2.5-flash로 변경해서 API를 호출하도록 코드를 수정해줘."

"AI 응답이 올 때까지 'AI가 여행 코스를 계획 중입니다'라는 로딩 메시지를 띄우고, 응답이 오면 삭제하는 로직을 추가해줘."

📂 Phase 4. UI/UX 디자인 (CSS Styling)
Figma 시안처럼 예쁘게 꾸미는 단계입니다.

"Figma 시안처럼 메인 컬러를 파란색(#007bff)으로 잡고, 전체적인 CSS 변수(:root)를 설정해줘."

"카카오톡처럼 사용자는 오른쪽, AI는 왼쪽에 말풍선이 뜨도록 CSS Flexbox를 활용해 스타일링해줘."

"헤더 왼쪽에 로봇 모양의 SVG 아이콘을 넣고 싶어. 깔끔한 벡터 아이콘 코드를 찾아줘."

"전송 버튼을 글자 대신 '종이비행기' 모양 아이콘으로 바꾸고 싶어. SVG 코드를 버튼 안에 넣어줘."

"채팅창의 스크롤바는 숨기거나 예쁘게 다듬고, 새 메시지가 오면 자동으로 맨 아래로 스크롤되게 해줘."

"입력창(input)의 테두리를 둥글게(border-radius) 만들고, 포커스 되었을 때 파란색 테두리가 생기게 해줘."

"AI 말풍선은 흰색 카드 형태에 그림자(box-shadow)를 줘서 입체감을 살려줘."

"웹 브라우저에서 볼 때 화면이 너무 넓어. 768px 이상 화면에서는 최대 너비를 800px로 제한하고 중앙 정렬해줘."

"데스크톱에서 버튼에 마우스를 올렸을 때 색상이 약간 진해지는 호버 효과를 추가해줘."

"웹 브라우저용 커스텀 스크롤바를 만들어줘. 회색 계열로 깔끔하게 스타일링해줘."

📂 Phase 5. 즐겨찾기 기능 구현 (Favorites Feature)
AI 추천을 저장하고 관리하는 단계입니다.

"AI 답변 말풍선 하단에 '즐겨찾기' 버튼을 추가하고, 클릭하면 해당 답변을 저장하는 기능을 만들어줘."

"localStorage를 사용해서 즐겨찾기한 추천 코스를 영구적으로 저장하는 코드를 작성해줘."

"즐겨찾기 데이터는 질문, 답변, 저장 시간을 포함한 객체 배열로 관리하고 싶어. 데이터 구조를 제안해줘."

"헤더에 하트 아이콘과 즐겨찾기 개수를 표시하는 버튼을 추가해줘. 클릭하면 즐겨찾기 화면으로 이동하게 해줘."

"채팅 화면과 즐겨찾기 화면을 전환할 수 있는 탭 UI를 만들어줘. 활성화된 탭은 파란색으로 강조되게 해줘."

"즐겨찾기 목록을 카드 형태로 보여주고, 각 카드에는 질문, 답변, 저장 시간이 표시되게 해줘."

"즐겨찾기 목록이 비어있을 때 '아직 즐겨찾기한 추천이 없습니다'라는 안내 메시지와 하트 아이콘을 중앙에 표시해줘."

"각 즐겨찾기 카드에 삭제 버튼을 추가하고, 클릭하면 확인창을 띄운 후 삭제하는 기능을 만들어줘."

"같은 답변을 다시 즐겨찾기하면 저장이 아니라 해제되도록 토글 기능을 구현해줘."

"즐겨찾기 버튼이 눌렸을 때 배경색을 노란색(#ffc107)으로 바꾸고 '즐겨찾기됨' 텍스트를 표시해줘."

"저장 시간을 '11월 29일 14:30' 형식으로 한국어로 보여주는 포맷 함수를 만들어줘."

"XSS 공격을 방지하기 위해 사용자 입력과 저장된 텍스트에 HTML 이스케이프 처리를 추가해줘."

"즐겨찾기 탭으로 전환하면 입력창이 숨겨지게 해줘. 채팅 탭으로 돌아오면 다시 보이게 해줘."

📂 Phase 6. 오류 수정 및 디테일 (Refining)
완성도를 높이는 마무리 단계입니다.

"사용자가 아무것도 입력하지 않고 전송 버튼을 눌렀을 때, 함수가 실행되지 않도록 예외 처리를 해줘."

"데이터 로딩 중에는 입력창과 전송 버튼을 비활성화(disabled) 했다가, 로딩이 끝나면 활성화하는 로직을 넣어줘."

"Gemini API 호출 중 에러가 발생하면, 콘솔에만 띄우지 말고 채팅창에 빨간 글씨로 '오류 발생' 메시지를 보여줘."

"AI 답변에 포함된 줄바꿈 문자(\n)를 HTML의 <br> 태그로 변환해서 줄바꿈이 적용되도록 해줘."

"AI가 답변에서 굵은 글씨(강조)를 쓰면 HTML <b> 태그로 변환해서 보여주는 정규표현식을 추가해줘."

"모바일에서도 화면이 깨지지 않도록 반응형 뷰포트 설정을 다시 확인해줘."

"API 키 공백 때문에 에러가 나는 것 같아. 키 값을 가져올 때 .trim() 함수를 적용해줘."

"body에 overflow: hidden을 추가해서 전체 화면 스크롤이 생기지 않게 해줘."

"모든 input과 button에 font-family를 명시적으로 지정해서 폰트가 일관되게 보이도록 해줘."

"긴 텍스트가 카드 밖으로 삐져나가지 않게 word-break: break-word를 추가해줘."

"전체 코드를 하나의 HTML 파일로 합쳐서 실행 가능하게 만들어줘."

"마지막으로 코드의 가독성을 위해 주석(Comment)을 달아서 설명해줘."
