 2일차: 기술 구조 확정 (Meeting 2: Technical Deep Dive)

 API 및 핵심 기능 점검

허윤: index.html을 보니 Promise.all로 API 3개를 동시에 호출하는 방식이 오류 처리까지 잘 되어 있더라고요.

const [attractionRes, foodRes, festivalRes] = await Promise.all([...]);


여기서 데이터를 파싱해서 allBusanData에 넣고, 그 JSON 데이터를 프롬프트에 통째로 넣어 RAG처럼 "데이터 기반 추천"이 동작하게 하는 구조가 맞죠?

김성보: 네, 정확합니다. JSON을 통째로 프롬프트에 넣어 컨텍스트를 제공하는 방식이에요. Gemini API URL도 이 방식을 그대로 사용합니다: https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent

허윤: 좋습니다. 그럼 저는 전체 기능 동작을 확인하고 코드를 수정하는 작업을 맡겠습니다. 특히 즐겨찾기 기능이 토글(Toggle) 방식으로 구현되어 있으니 테스트할 부분이 많겠네요.
